## üîß TROUBLESHOOTING

### ‚ö†Ô∏è "Audio file not found"

**Problem**: You see a red warning about the audio file

**Solutions**:
1. Make sure `Crucible_1.mp3` is in the **same folder** as the HTML file
2. Make sure it's spelled exactly: `Crucible_1.mp3` (capital C, underscore, no spaces)
3. Try renaming your audio file to match exactly

### ‚ö†Ô∏è PDF doesn't show

**Problem**: The PDF area is blank or shows an error

**Solutions**:
1. Make sure the PDF is in the same folder
2. Make sure it's named exactly: `07_Crucible_Act_I_Complete.pdf`
3. Try opening the HTML file in a different browser
4. Some browsers block PDFs - try Chrome or Firefox

### ‚ö†Ô∏è Play button doesn't work

**Problem**: Clicking play does nothing

**Solutions**:
1. Try clicking directly on the audio player's play button (not the button below it)
2. Check your browser's address bar for a "blocked audio" icon - click it to allow
3. Right-click the audio player and select "Play"
4. Refresh the page (F5 or Cmd+R)

### ‚ö†Ô∏è Scenes don't jump to the right page

**Problem**: PDF stays on the same page when you click scenes

**Solutions**:
1. This is normal for some browsers - they restrict iframe navigation
2. You can manually scroll the PDF to the right page
3. The page number is shown in the bottom-right corner
4. Try using Chrome or Firefox which handle this better

## üìù ADJUSTING TIMESTAMPS

The timestamps are estimated. To make them perfect for your audio:

1. **Listen to your audio file** and note when each scene begins
2. **Open the HTML file** in a text editor (Notepad, TextEdit, VS Code)
3. **Find this section** (around line 250):

```javascript
const scenes = [
    { id: 1, name: "Opening / Overture", time: 0, page: 4 },
    { id: 2, name: "Parris and Betty", time: 120, page: 4 },
    // etc...
];
```

4. **Change the `time:` values** to match your audio (time is in seconds)
5. **Change the `page:` values** if needed to match your PDF
6. **Save the file** and refresh your browser
